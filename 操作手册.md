# DowokerSystem Node.js 后端脚手架操作手册

## 一、项目概述

这是一个基于 **Express + TypeScript + MongoDB** 的 Node.js 后端脚手架，内置用户认证系统，可快速搭建新的后端项目。

### 核心技术栈

| 技术               | 版本  | 用途     |
| :----------------- | :---- | :------- |
| Express            | 5.2.1 | Web 框架 |
| TypeScript         | 6.0.2 | 类型安全 |
| MongoDB + Mongoose | 9.3.3 | 数据库   |
| JWT                | 9.0.3 | 身份认证 |
| bcryptjs           | 3.0.3 | 密码加密 |

## 三、快速开始

### 3.1 克隆脚手架

# 1. 复制项目文件夹
cp -r DowokerSystem my-new-project

# 2. 进入项目目录
cd my-new-project

# 3. 安装依赖
pnpm install

### 3.2 环境配置

编辑 `.env` 文件：

\# 服务端口

 PORT=5000

\# MongoDB 连接地址 

MONGO_URI=mongodb://127.0.0.1:27017/你的数据库名

\# JWT 密钥（生产环境请修改为复杂字符串）

JWT_SECRET=你的随机密钥

### 3.3 启动项目

```
bash# 开发模式（支持热重载）
pnpm dev

# 构建生产版本
pnpm build

# 启动生产服务
pnpm start
```

## 四、核心模块说明

### 4.1 认证模块（auth）

**文件位置**: `src/controllers/authController.ts`

| 接口 | 方法 | 路径                 | 说明           |
| :--- | :--- | :------------------- | :------------- |
| 注册 | POST | `/api/auth/register` | 创建新用户     |
| 登录 | POST | `/api/auth/login`    | 获取 JWT token |

**使用示例**:

```
typescript// 注册请求
POST /api/auth/register
{
  "username": "test",
  "password": "123456"
}

// 登录请求
POST /api/auth/login
{
  "username": "test",
  "password": "123456"
}
// 返回 { msg: '登录成功', token: 'xxx' }
```

### 4.2 认证中间件

**文件位置**: `src/middleware/auth.ts`

用于保护需要登录的接口：

```
typescriptimport authMiddleware from './middleware/auth';

// 在路由中使用
router.get('/profile', authMiddleware, getProfile);
```

**请求头格式**:

```
Authorization: Bearer <your_token>
```

### 4.3 数据模型

**文件位置**: `src/models/User.ts`

## 五、扩展指南

### 5.1 添加新路由

1. **创建控制器** `src/controllers/productController.ts`:

```
typescriptimport type { Request, Response } from "express";
import Product from "../models/Product";

export const getProducts = async (req: Request, res: Response) => {
  const products = await Product.find();
  res.json(products);
};
```

1. **创建路由** `src/routes/product.ts`:

```
typescriptimport express from 'express';
import { getProducts } from '../controllers/productController';
import authMiddleware from '../middleware/auth';

const router: express.Router = express.Router();

router.get('/', authMiddleware, getProducts);

export default router;
```

1. **注册路由** `src/index.ts`:

```
typescriptimport productRoutes from './routes/product';

app.use('/api/products', productRoutes);
```

### 5.2 添加新模型

1. 在 `src/models/` 下创建新模型文件
2. 定义 Schema 结构
3. 导出模型供控制器使用

### 5.3 添加新中间件

1. 在 `src/middleware/` 下创建中间件文件
2. 遵循 `(req, res, next)` 函数签名
3. 在路由中按需引入使用

## 七、最佳实践

### 7.1 代码规范

- ✅ 控制器只处理业务逻辑，不直接操作数据库
- ✅ 使用中间件统一处理认证、日志等横切关注点
- ✅ 密码必须加密存储，禁止明文
- ✅ 敏感配置放入 `.env` 文件，不提交到版本控制

### 7.2 安全建议

- 🔒 生产环境使用强 JWT 密钥
- 🔒 启用 HTTPS
- 🔒 添加请求频率限制
- 🔒 输入数据验证（可添加 `joi` 或 `zod`）

### 7.3 目录命名规范

| 类型   | 命名规范           | 示例                                    |
| :----- | :----------------- | :-------------------------------------- |
| 控制器 | `xxxController.ts` | [authController.ts](javascript:void(0)) |
| 模型   | `Xxx.ts` (大写)    | [User.ts](javascript:void(0))           |
| 路由   | `xxx.ts`           | [auth.ts](javascript:void(0))           |
| 中间件 | `xxx.ts`           | [auth.ts](javascript:void(0))           |

## 八、常见问题

### Q1: 数据库连接失败？

- 检查 MongoDB 服务是否启动
- 检查 `.env` 中 `MONGO_URI` 配置是否正确

### Q2: Token 验证失败？

- 检查请求头格式：`Authorization: Bearer <token>`
- 检查 `JWT_SECRET` 是否与生成时一致

### Q3: 跨域问题？

- 已内置 [cors](javascript:void(0)) 中间件，如需自定义配置可修改 `src/index.ts`
