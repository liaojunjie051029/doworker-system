# DowokerSystem API 接口文档

## 1. 认证接口

### 1.1 注册接口
- **路径**: `/api/auth/register`
- **方法**: `POST`
- **描述**: 创建新用户
- **请求参数**:
  | 参数名 | 类型 | 必填 | 描述 |
  |-------|------|------|------|
  | username | string | 是 | 用户名 |
  | password | string | 是 | 密码 |
- **响应示例**:
  ```json
  {
    "msg": "注册成功",
    "user": {
      "id": "190ec1e3edb2a82a015013ea",
      "username": "testuser"
    }
  }
  ```
- **错误响应**:
  ```json
  {
    "msg": "注册失败"
  }
  ```

### 1.2 登录接口
- **路径**: `/api/auth/login`
- **方法**: `POST`
- **描述**: 用户登录并获取 JWT token
- **请求参数**:
  | 参数名 | 类型 | 必填 | 描述 |
  |-------|------|------|------|
  | username | string | 是 | 用户名 |
  | password | string | 是 | 密码 |
- **响应示例**:
  ```json
  {
    "msg": "登录成功",
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
  ```
- **错误响应**:
  ```json
  {
    "msg": "用户不存在"
  }
  ```
  或
  ```json
  {
    "msg": "密码错误"
  }
  ```

## 2. 用户接口

> 所有用户接口都需要在请求头中添加 `Authorization: Bearer <token>`

### 2.1 获取用户列表
- **路径**: `/api/user/userlist`
- **方法**: `GET`
- **描述**: 获取所有用户列表

## 6. 认证说明

### 6.1 JWT Token
- 登录成功后，服务器会返回一个 JWT token
- 所有需要认证的接口都需要在请求头中添加 `Authorization: Bearer <token>`
- Token 有效期为 1 天

### 6.2 状态码说明
- `200`: 请求成功
- `400`: 请求参数错误
- `401`: 未授权（token 无效或过期）
- `500`: 服务器内部错误

## 7. 接口调用示例

### 7.1 使用 Postman 调用
1. 首先调用登录接口获取 token
2. 在请求头中添加 `Authorization: Bearer <token>`
3. 调用其他需要认证的接口

### 7.2 使用 Fetch API 调用
```javascript
// 登录获取 token
const loginResponse = await fetch('https://doworker-system-production.up.railway.app/api/auth/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    username: 'testuser',
    password: '123456'
  })
});

const loginData = await loginResponse.json();
const token = loginData.token;

// 使用 token 调用其他接口
const workerListResponse = await fetch('https://doworker-system-production.up.railway.app/api/worker/workerList', {
  headers: {
    'Authorization': `Bearer ${token}`
  }
});

const workerList = await workerListResponse.json();
console.log(workerList);
```

## 8. 注意事项
- 所有非认证接口（注册、登录）不需要 token
- 所有认证接口需要在请求头中添加 token
- 密码会在服务器端进行加密存储，不会明文存储
- 项目使用 MongoDB 数据库，所有数据会自动创建集合
- 接口返回的数据结构可能会根据实际情况有所调整
